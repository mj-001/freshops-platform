# Contributing to FreshOpsPlatform

First off, thank you for considering contributing to FreshOpsPlatform! It's people like you that make this open-source suite exceptional.

## Code of Conduct

By participating in this project, you agree to abide by our Code of Conduct. Please maintain professional, friendly, and respectful communication.

## How to Contribute

### 1. Reporting Bugs
- Search existing issues or discussions to ensure the bug hasn't been reported.
- Open a new issue with a clear description, reproduction steps, expected behavior, actual behavior, and relevant logs or screenshots.

### 2. Suggesting Enhancements
- Open a discussion or issue describing the feature you'd like to see, why it is useful, and how it fits into the FreshOpsPlatform capabilities.

### 3. Pull Requests
- Fork the repository.
- Create a feature branch (`git checkout -b feature/amazing-feature`).
- Ensure all TypeScript code is non-empty, type-safe, and passes compilation (`npm run build`).
- Ensure tests still pass.
- Write meaningful commit messages.
- Submit a Pull Request targeting the `main` branch.

## Development Setup

1. Install dependencies:
   ```bash
   npm install
   ```

2. Run development server:
   ```bash
   npm run dev
   ```

3. Build and compile:
   ```bash
   npm run build
   ```
